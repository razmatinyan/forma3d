# Architecture

Where everything lives and why.

## Content model

The single most important convention:

- **`i18n/locales/*.json`** holds everything a human reads.
- **`app/data/*.ts`** holds identity, ordering, and structure: keys, icon names, hrefs, slugs, layout classes.

A component reads both and hardcodes neither. This is what lets copy change without touching Vue files, and structure change without touching translations.

```
courses = [{ key: 'archviz', slug: '...', icon: '...', status: 'soon' }]   // data
courses.items.archviz.title = "Architectural Visualization"                 // i18n
```

## Directory map

```
app/
  app.vue                  root: locale head, global [data-reveal] batch
  layouts/default.vue      backdrop + header + main + footer
  pages/
    index.vue              home, composes the section components
    courses/[slug].vue     course detail, 404s on unknown slug
    privacy.vue            privacy policy, linked from the consent checkbox
  components/
    site/                  shell: header, footer, brand, mobile nav, lang switcher
    section/               one file per page section, owns its own layout
    common/                reusable pieces: frames, buttons, cards, rows
    form/                  RegistrationForm
    ui/                    shadcn-vue generated, do not hand-edit
  composables/
    useGSAP.ts             the only entry point for animation
    useJsonLd.ts           structured data for the home page
    useLocaleFormat.ts     hydration-safe Intl date and number formatting
  data/
    site.ts                PLACEHOLDER business data, replace before launch
    course.ts              courses, modules, mentors, faq, contact rows
    nav.ts                 header and footer nav links
  plugins/
    scroll-refresh.client.ts  ScrollTrigger.refresh() after fonts load
shared/                    imported by BOTH app and server
  registration.ts          zod schema, the single source of validation truth
  emailCopy.ts             per-locale email strings
server/
  api/register.post.ts     validate, reject honeypot, rate limit, send two emails
  emails/*.vue             notification (to admin) and confirmation (to applicant)
```

## Page composition

`index.vue` renders sections in order. Each `Section*.vue` owns its own grid, its own scroll animation, and wraps itself in `FrameSection`.

| Order | Component | Anchor |
| --- | --- | --- |
| 1 | `SectionHero` | `#hero` |
| 2 | `SectionCourses` | `#courses` |
| 3 | `SectionSignals` | `#facts` |
| 4 | `SectionProtocol` | `#how` |
| 5 | `SectionFaq` | `#faq` |
| 6 | `SectionCta` | `#cta` |
| 7 | `SectionContact` | `#contact` |
| 8 | `SectionRegister` | `#register` |

Course pages use `#overview`, `#pros`, `#program`, `#mentor`, `#register`. The `#program` block only renders for `3ds-max`, since the other two courses have no modules defined yet.

Anchor targets need `scroll-mt-24` because the header is sticky.

## Naming

`common/` is registered with `pathPrefix: false` in `nuxt.config.ts`, so `common/MetricCard.vue` is `<MetricCard>` rather than `<CommonMetricCard>`. Everything else keeps path prefixing: `form/RegistrationForm.vue` is `<FormRegistrationForm>`. `site/SiteHeader.vue` is `<SiteHeader>` because the directory and filename already share a segment. See `docs/nuxt.md` for the full table.

## Registration flow

1. `RegistrationForm` validates client-side against `shared/registration.ts`
2. Posts to `/api/register` with the current locale
3. Server re-validates with the same schema, rejects honeypot hits with 422, rate limits at 429
4. Sends an admin notification and an applicant confirmation
5. A failed confirmation still returns success, because the lead is already captured. Only a failed admin notification returns an error.

The applicant email renders in the locale they submitted from. `useI18n()` does not exist in a Nitro handler, so strings come from `shared/emailCopy.ts` as pre-resolved props.

## Motion

Everything routes through `useGSAP()`. See `DESIGN.md` for the presets.

Two entry points:
- `motion(queries, fn, scope)` for media-query scoped animation, always reduced-motion aware
- `revealFrom` / `revealTo` shared presets so blocks animate consistently

`data-reveal` marks a **container**; the global batch in `app.vue` animates its **children**. Cards never move, because moving a card moves its borders and the borders are what make a section read as one panel.

## Third-party

`app/components/ui/` is generated by the shadcn-vue CLI. Treat it as vendored: themed once to the palette, not hand-edited beyond that. Nine primitives are installed; `button` is a transitive dependency of `sheet` and is not used directly. `AppButton` is the real button.
