# i18n

Three locales. Armenian is the default and unprefixed, Russian is `/ru`, English is `/en`.

More bugs have come from this area than any other. Read before touching a string.

## Config facts

- `defaultLocale: 'hy'`, `strategy: 'prefix_except_default'`
- `detectBrowserLanguage: false` on purpose. With it on, `redirectOn: 'root'` sent every English-preferring browser from `/` to `/en` and the cookie made it stick.
- The locale property is `language`, **not** `iso`. `@nuxtjs/i18n` v10 renamed it and silently ignores the old name, which kills the `lang` attribute and every hreflang tag.
- `baseUrl` must be set or alternate links cannot be generated.

## Adding a string

1. Add the key to **all three** files in `i18n/locales/`.
2. Reference it as `t('some.key')`, never as literal text.
3. Verify parity before committing.

```bash
node -e "
const fs=require('fs');
function keys(o,p,a=new Set()){for(const k in o){const v=o[k];const n=p?p+'.'+k:k;if(typeof v==='string')a.add(n);else if(v&&typeof v==='object')keys(v,n,a)}return a}
const en=keys(JSON.parse(fs.readFileSync('i18n/locales/en.json','utf8')),'');
for(const l of ['ru','hy']){const s=keys(JSON.parse(fs.readFileSync('i18n/locales/'+l+'.json','utf8')));const m=[...en].filter(k=>!s.has(k));console.log(l+': '+s.size+'/'+en.size+(m.length?' MISSING '+m.join(', '):' ok'))}
"
```

## Writing translations

Write real Armenian and Russian, not transliterated English. If unsure of a phrase, prefer simple and correct over florid and wrong.

**Armenian punctuation differs.** The full stop is `։` (U+0589), the question mark is `՞` and sits inside the word. A regex looking for `.` will silently skip Armenian.

**Armenian and Russian run 20 to 40 percent longer than English.** Buttons, nav items, and card titles are the pressure points. Check 375px in all three languages.

**Armenian opts out of tight tracking** via `html:lang(hy) h1,h2,h3 { letter-spacing: normal }`. Armenian letterforms break down under negative tracking.

**Product names stay English everywhere.** `3D's Max`, `Adobe Photoshop`, `Architectural Visualization`, `Game Dev Modeling`.

## Linking

Always `localePath()`:

```vue
<NuxtLink :to="localePath('/#courses')">      <!-- correct -->
<NuxtLink :to="localePath(`/courses/${slug}`)">
<a href="#courses">                            <!-- breaks on /courses/* and ignores locale -->
```

## Interpolation and slots

Values go through named params, not string concatenation:

```json
{ "mentorYears": "{years} years working in the field" }
```

When part of a sentence must stay untranslated or become a link, use `<i18n-t>` with a slot. `scope="global"` is required or vue-i18n warns about a missing parent scope:

```vue
<i18n-t keypath="footer.developedBy" tag="span" scope="global">
  <template #author><a href="...">Razmik Matinyan</a></template>
</i18n-t>
```

Armenian often needs a case suffix after such a slot, for example `{author}-ի կողմից`.

## Formatting numbers and dates

Never call `Intl` directly in a component. Use `useLocaleFormat.ts`:

```ts
const label = useLocaleDate(site.course.nextStartDate, { day: 'numeric', month: 'long' }, 'heroStart')
const price = useLocaleNumber(site.course.priceAmd, 'price')
```

Direct `Intl` in a `computed()` runs on the server and again on the client, and the two can resolve different locales. That produced English dates overwriting Armenian ones during hydration. The composable serializes the server result into the payload and keys it by locale so switching language still reformats.

## Emails

Nitro handlers have no `useI18n()`. Email strings live in `shared/emailCopy.ts`, keyed by locale, and are passed to the template as pre-resolved props. The client sends its locale with the submission and the server validates it against the three known codes.
